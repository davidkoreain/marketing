# 제품 기획안 (PRD: Product Requirements Document)

## 1. 프로젝트 개요
* **프로젝트명**: 소상공인 초간편 SNS 마케팅 자동화 서비스 (가칭: **소마비 - SoMaBi**)
* **목표**: 마케팅 여력이 부족한 소상공인들이 상품 정보나 키워드만 입력하면, AI가 SNS 홍보글, 이미지, 쇼츠 영상을 자동으로 생성하고 카카오톡/인스타그램 채널로 원클릭 배포할 수 있는 서비스 구축.
* **핵심 차별점**: 무조건적인 자동 생성이 아닌, **단계별 피드백 루프(Human-in-the-loop)**를 도입하여 소상공인이 글, 이미지, 영상을 직접 검토 및 수정하고 최종 승인된 콘텐츠만 배포되도록 보장함. 이를 위해 **LangGraph**를 워크플로우 엔진으로 채택함.

---

## 2. 타겟 고객 (Target Audience)
1. **1인 자영업자 및 소상공인**: 카페, 식당, 꽃가게, 의류 매장 등을 운영하며 인스타그램이나 카카오톡 채널 관리에 부담을 느끼는 분들.
2. **초기 스타터/쇼핑몰 창업자**: 매일 반복되는 홍보 콘텐츠 제작 시간을 줄이고 본업에 집중하고자 하는 창업자.
3. **디지털 마케팅 초보자**: 디자인 도구나 영상 편집 프로그램을 다루지 못해 고품질 콘텐츠 제작이 어려운 사용자.

---

## 3. 핵심 기능 요구사항 (Key Features)

### 3.1. 마케팅 입력 수집 (Input Stage)
* **기능**: 사용자가 홍보하려는 상품이나 이벤트의 기본 정보를 입력받음.
* **입력 항목**:
  * 상품명 및 설명 (또는 관련 이미지 첨부)
  * 핵심 키워드 (예: #감성카페, #신메뉴출시)
  * 타겟 고객층 (예: 20대 여성, 직장인)
  * 브랜드 톤앤매너 (예: 친근한, 전문적인, 감성적인)
  * 배포 채널 선택 (인스타그램 피드/스토리, 카카오톡 채널 알림톡 등)

### 3.2. LangGraph 기반 단계별 콘텐츠 생성 및 피드백 루프 (Core Workflow)
LangGraph를 활용하여 생성 과정을 노드(Node)화하고, 각 단계마다 사용자 피드백을 대기(`interrupt`)하는 구조를 설계합니다.

```mermaid
graph TD
    Start([1. 사용자 정보 입력]) --> GenText[2. SNS 홍보글 생성]
    GenText --> IntText{사용자 피드백 대기}
    IntText -- 수정 요청 / 반려 -- > GenText
    IntText -- 승인 --> GenImg[3. 홍보 이미지 생성]
    
    GenImg --> IntImg{사용자 피드백 대기}
    IntImg -- 수정 요청 / 반려 --> GenImg
    IntImg -- 승인 --> GenVideo[4. 쇼츠 영상 생성]
    
    GenVideo --> IntVideo{사용자 피드백 대기}
    IntVideo -- 수정 요청 / 반려 --> GenVideo
    IntVideo -- 승인 --> Publish[5. 카카오톡/인스타 배포]
    
    Publish --> End([6. 배포 완료 및 분석])
```

#### ① [1단계] SNS 홍보글 생성 (Node 1)
* **설명**: LLM을 통해 인스타그램 해시태그가 포함된 트렌디한 본문 글과 카카오톡 채널 맞춤형 알림 메시지를 생성합니다.
* **피드백**: 사용자는 AI가 작성한 글을 직접 수정하거나, "더 감성적인 톤으로 수정해 줘" 등의 자연어 피드백을 입력하여 재생성할 수 있습니다.

#### ② [2단계] 홍보 이미지 생성 (Node 2)
* **설명**: 확정된 홍보글의 맥락과 설정된 톤앤매너에 맞는 홍보 이미지를 생성합니다.
* **피드백**: 이미지의 구도, 포함될 요소 등에 대해 재생성을 요청하거나 직접 다른 이미지를 업로드하여 대체할 수 있습니다.

#### ③ [3단계] 쇼츠 영상 생성 (Node 3)
* **설명**: 홍보글과 이미지를 기반으로 숏폼 영상(대본 작성 -> 성우 TTS 음성 합성 -> 배경음악 선정 -> 이미지 슬라이드/비디오 클립 합성)을 자동 생성합니다.
* **피드백**: 자막 내용 수정, 성우 목소리(TTS) 변경, 영상 템플릿 변경 등을 요청할 수 있습니다.

#### ④ [4단계] 자동 배포 및 예약 (Node 4)
* **설명**: 최종 승인된 글, 이미지, 영상을 인스타그램 API와 카카오 비즈니스 API를 통해 즉시 배포하거나 원하는 시간에 예약 배포합니다.

## 4. 비기능적 요구사항 (Non-functional Requirements)
* **반응형 웹 UI/UX**: 소상공인이 매장에서 스마트폰으로 간편하게 관리할 수 있도록 **모바일 우선(Mobile-First)** 반응형 디자인 적용.
* **상태 보존(State Persistence)**: 피드백 중 이탈하더라도 작성 중이던 콘텐츠와 진행 단계가 유지되어야 함 (LangGraph Memory Saver 활용).
* **비동기 처리**: 이미지 및 동영상 생성은 시간이 다소 소요되므로, 백그라운드 큐(Queue) 처리 및 진행 상태 실시간 UI 업데이트(WebSocket 또는 Server-Sent Events) 지원.

---

## 5. 단계별 프로젝트 마일스톤
* **Phase 1**: 기획 및 기술 검증 (PoC)
  * PRD 및 기술 스택 확정
  * LangGraph를 활용한 텍스트/이미지 피드백 워크플로우 백엔드 프로토타입 검증
* **Phase 2**: 핵심 기능 개발 (MVP)
  * 프론트엔드 UI 구축 (Next.js) 및 백엔드 API 연동
  * LangGraph Human-in-the-loop 인터랙션 구현 (글 & 이미지 단계)
  * 카카오톡/인스타 API 연동 기초 작업
* **Phase 3**: 미디어 확장 및 고도화
  * 쇼츠 영상 렌더링 엔진 연동 (대본 + TTS + 이미지 합성)
  * 예약 배포 기능 및 배포 결과 트래킹
* **Phase 4**: 안정화 및 출시
  * 실사용자 테스트 (소상공인 FGI) 및 UI/UX 피드백 반영
  * 모바일 최적화 및 보안 검토 후 정식 배포
